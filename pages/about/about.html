The Full Story

My introduction to web content creation technically began in 2006 when I was in the fourth grade. I wanted my homeroom to keep in touch outside of the classroom, and given that cellphones weren't as widespread, I started an online discussion board by leveraging a free hosting service. It was fun, and surpsingly taught me how to Google for problems and solutions at an early age.

But my real passion for software engineering began at Virginia Tech, where I was first introduced to programming. I vividly remember sitting in my chair in chilly Blacksburg, Virginia — meticulously reading my Java textbook with blankets and a hoodie — being in total awe with the analgous nature of object oriented programming to physical objects in real life. It was the first time in a long time where things in school just started to make sense. It took off from there, and my programming classes were starting to feel more like something I would do for fun and not for a grade. Eventually, when graduation rolled around, I realized that I was most interested in engineering the graphical user interface (GUI) for interfacing with other web APIs. I like to think it has to do with my artistic background — such that instead of starting with a white piece of paper, I traded it for a blank webpage.

Since then, I've supported a variety of web applications as a front end engineer. I'm especially knowledable in building composable user interface components with React to support design, accessibiliy, and performance patterns. I like to read, which bodes particularly well in the fast paced JavaScript front end ecoystem; so I'm never afraid to try something new if the tool fits the job.

However, the most valuable lesson I've learned is not a technical one, but rather, one focused on empowering others. I've learned to instill empathy as a daily driver in all that I do; if I can make the experience better for those around me, it will help the applications that I contribute to holistically. Be it making the distributed application a smaller file size, priorizing accessibility, or leaving traceable comments and documentation to help my teammates, I understand that I am not an engineer of one. It is the users and team around me that keep my software engineering goals and aspirations alive.

Brennan Arvo
